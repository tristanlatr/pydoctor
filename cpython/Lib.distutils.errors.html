distutils.errors Provides exceptions used by the Distutils modules. Note that Distutils modules may raise standard exceptions; in particular, SystemExit is usually raised for errors that are obviously the end-user's fault (eg. bad command-line arguments). This module is safe to use in "from ... import *" mode; it only exports symbols whose names start with "Distutils" and end with "Error".

Class DistutilsError The root of all Distutils evil.
Class DistutilsModuleError Unable to load an expected module, or to find an expected class within some module (in particular, command modules and classes).
Class DistutilsClassError No summary
Class DistutilsGetoptError The option table provided to 'fancy_getopt()' is bogus.
Class DistutilsArgError Raised by fancy_getopt in response to getopt.error -- ie. an error in the command line usage.
Class DistutilsFileError Any problems in the filesystem: expected file not found, etc. Typically this is for problems that we detect before OSError could be raised.
Class DistutilsOptionError No summary
Class DistutilsSetupError For errors that can be definitely blamed on the setup script, such as invalid keyword arguments to 'setup()'.
Class DistutilsPlatformError We don't know how to do something on the current platform (but we do know how to do it on some platform) -- eg. trying to compile C files on a platform not supported by a CCompiler subclass.
Class DistutilsExecError Any problems executing an external program (such as the C compiler, when compiling C files).
Class DistutilsInternalError Internal inconsistencies or impossibilities (obviously, this should never be seen if the code is working!).
Class DistutilsTemplateError Syntax error in a file list template.
Class DistutilsByteCompileError Byte compile error.
